# Incident Playbook

## Rollback
- Roll back to previous stable release in case of critical issues.
- Notify stakeholders and users about the rollback.

## Crash Triage
- Monitor crash reports from opt-in crash reporting.
- Prioritize crashes by frequency and severity.
- Reproduce issues locally or in staging.

## Hotfix
- Branch from latest stable release.
- Fix critical bugs.
- Run full test suite.
- Deploy hotfix build.
- Monitor post-hotfix stability.

## Communication
- Maintain clear communication channels with users and stakeholders.
- Provide timely updates on incidents and resolutions.

## Documentation
- Document all incidents, root causes, and fixes.
- Update playbook based on learnings.
